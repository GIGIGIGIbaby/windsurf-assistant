## 一 · bindings.json schema

### 1.1 顶层

```json
{
  "version": "1.0.0",
  "savedAt": "ISO8601",        ← 上次写盘时
  "lastSyncedAt": "ISO8601",   ← 上次 verify (探活) 时
  "bindings": [ ... ],         ← 真活之绑定列
  "history": [ ... ]           ← unbind / rebind 之 record (守玄德)
}
```

### 1.2 binding 单件

```json
{
  "idx": 0,                                      ← 序号 (主公分流之锚)
  "account": {
    "email": "...@gmail.com",                    ← windsurf 邮箱
    "tag": "c7d9bf15",                           ← JWT exp 标 (8 hex)
    "tokenLast4": "...a1b2",                     ← token 末 4 (避全文)
    "source": "unify | wam | accounts | tokens"  ← 来源 (vm_unify pick 默 unify)
  },
  "vm": {
    "sessionId": "devin-xxxx",                   ← Devin VM session id (唯一)
    "tunnelUrl": "https://omni-router-app-tunnel-XXX.devinapps.com",
    "spawnedAt": "ISO8601",                      ← spawn 启时
    "deployedAt": "ISO8601",                     ← dao_proxy 装完时
    "ttlExpire": "ISO8601",                      ← spawnedAt + 24h
    "deployStatus": "pending | deployed | failed",
    "spStrategy": "bypass | override | prepend | append | dao | custom | usernote",
    "spConfig": "<sp-config-file-path>",         ← 若用 _sp_configs/*.json
    "spCustom": "<custom SP 文>",                ← strategy=override/custom
    "spDaoChapter": 22,                          ← strategy=dao
    "spNote": "<usernote 文>",                   ← strategy=usernote
    "metaDeployStatus": "not_deployed | deployed | failed",  ← :8081 meta_router 状
    "publicTunnelUrl": "https://*.trycloudflare.com"  ← 印 125 · 公网入口备路 (可选)
  },
  "boundAt": "ISO8601",         ← 此 binding 立时
  "lastVerifiedAt": "ISO8601",  ← 上次 /health 探时
  "status": "alive | dead | unbound | rebinding",
  "memo": "主公此号此 VM 之意图说明 (人读)"
}
```

### 1.3 history 单件 (unbind / rebind 之 record)

```json
{
  "idx": 99,
  "account": { "email": "...", "tag": "..." },
  "vm": { "sessionId": "...", "tunnelUrl": "..." },
  "unboundAt": "ISO8601",
  "reason": "main_unbound_by_user | ttl_expired_no_rebind | account_dead | manual_swap",
  "boundDurationHours": 23.7,
  "_comment_": "守玄德 · 不弃旧 record · 主公自决"
}
```

## 二 · 一号一VM.ps1 之 actions 全表

| Action | 0 ACU? | 何为 |
|--------|--------|------|
| `plan` | ✓ | 0 ACU · 调 vm_unify pick --count N · 显将选哪 N 号 + 各号会启何 VM |
| `go` | ✗ | ~N ACU · 真起 N 件 · 各号一件 · 写 bindings.json |
| `wait` | ✓ | 0 ACU · poll 各 VM 之 /health · 等齐起 (~10 min) |
| `status` | ✓ | 0 ACU · 列 bindings.json 之全 · 含 alive/dead 状 |
| `verify` | ✓ | 0 ACU · 探每件 binding 之 /health + dao_proxy + meta_router |
| `set-sp` | ✓ | 0 ACU · 改 binding 之 spStrategy · 重 deploy dao_proxy · 写 bindings |
| `rebind` | ✗ | ~1 ACU · 旧 VM 死 (TTL 过) · 用同号起新 VM · 续绑 (旧入 history) |
| `unbind` | ✓ | 0 ACU · 主公主动解绑 · 标 status=unbound · 留 record |
| `export` | ✓ | 0 ACU · print bindings.json 至 stdout (主公备份) |
| `import` | ✓ | 0 ACU · 从 stdin / -File 读 bindings · 合并 (主公迁机) |

## 三 · 故障与修

| 状 | 真因 | 修 |
|---|------|----|
| `plan` 报 0 fresh | unify 三源全 used/dead | 主公等账重置 (D 1.1h · W 1.1h) 或加新号至 ~/.dao/accounts.json |
| `go` 第 N 件 spawn 失败 | 此号已有 alive VM 24h 内 (1 号 1 VM/24h) | 此号已绑 · 跳之 (idempotent) · 或 -Force 杀旧起新 |
| `wait` 超时 (>10 min) | spawn 之 wss 慢 / Devin 服反 | 看 _state/vm_pool.json + watchdog.log · 若 spawn ok 但 deploy 失则 set-sp 重 deploy |
| `verify` 报某 binding dead | VM TTL 过期 (24h) | `.\一号一VM.ps1 rebind -Idx N` 用同号起新 VM 续绑 |
| `bindings.json` 损 (json 解析失败) | 主公手编误 / 写盘时崩 | 自动 fallback 至 `bindings.json.bak`(每次写盘前 cp) · 或 plan 之 `--reset` 重立 |
| 同 idx 之 binding 重复 | 主公并发跑 一号一VM.ps1 | 文件锁 (Win32 LockFileEx)·若仍重则手编 + import |
| `rebind` 后旧 sessionId 仍 alive | 24h 内 + 旧 VM 未死 (撕裂态) | 旧 VM TTL 自然死 (24h) · 不强 stop · 守 SLA |

## 四 · 持久化之三守

```text
✓ 写盘原子 · tmp 写完 → rename (Move-Item) · 不破坏旧件
✓ 写盘备份 · bindings.json.bak (上次成果) · 主公可一字回滚
✓ 写盘历       · _logs/bindings_YYYYMMDD_HHMMSS.json (历代 snapshot)
```

## 五 · 与 vm_pool.json 之分

`bindings.json` (此目) 与 `00_本源/_state/vm_pool.json` (印 119 立) **互不依**:

| 表 | 持有者 | 何为 | 何用 |
|---|--------|------|------|
| `vm_pool.json` | vm_omni / vm_pool_watchdog | 真活 VM 池态 (alive/dead 真探) | watchdog 5min poll · 自换之 |
| `bindings.json` | 一号一VM.ps1 (此目) | 号-VM 之 1:1 绑定意图 | 主公生产 · 长跑 · 持久 |

**verify** action 自动 cross-check 二表：
- bindings 中 sessionId 不在 pool · 标 dead (VM 真已死)
- pool 中 sessionId 不在 bindings · 该 VM 非此 1:1 绑定生 (e.g., spawn_N.ps1 起的) · 不动
