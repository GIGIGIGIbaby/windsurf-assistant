## 一 · SP 七态全表

| 态 | 何时用 | 主公配 (本地 ENV) |
|----|--------|------------------|
| `bypass` | 默 · 任 client 之 system 完全透 | `$env:SP_STRATEGY = "bypass"` |
| `override` | 强统一 SP · 客 SP 全弃 | `$env:SP_STRATEGY = "override"; $env:SP_CUSTOM = "..."` |
| `prepend` | dao 加在 client SP 之前 | `$env:SP_STRATEGY = "prepend"` |
| `append` | client SP 之后加 dao | `$env:SP_STRATEGY = "append"` |
| `dao` | silk 帛书 N 章 (随机或 pin) | `$env:SP_STRATEGY = "dao"; $env:SP_DAO_CHAPTER = "22"` |
| `custom` | 主公自定文 (从 env) | `$env:SP_STRATEGY = "custom"; $env:SP_CUSTOM = "你是..."` |
| `usernote` | **印 122 立** · SP §3.17 合法槽 | `$env:SP_STRATEGY = "usernote"; $env:SP_NOTE = "..."` |

ENV 之配在**部署时**生效——主公启 `vm_proxy_deploy.js` 时之 ENV 自动嵌入 VM 之 dao_proxy 启动脚本。


### 2.2 法 2 · 注 dao_proxy 之 endpoint (热换)

```pwsh

```

### 2.3 法 3 · per-request header (主公 SDK 自定)


### 2.4 法 4 · _sp_configs/{vm-name}.json + --sp-config (★ 印 125 · per-VM 隔离)

**真意**: 主公一字编 `00_本源/_sp_configs/{vm-name}.json` · 部署时引用 · 一 VM 一 SP 态 · per-VM 隔离不犯。

#### 2.4.1 schema

```json

  "name": "vm-甲",
  "strategy": "bypass | override | prepend | append | dao | custom | usernote",

  "dao_chapter": 22,
  "neutralize": true,
  "strip_mem": true,
  "strip_side": true,
  "note": "(strategy=usernote 时之 user note 文)",
  "memo": "主公此 VM 之意图说明 (人读)"
```

#### 2.4.2 用 (主公一字)

```pwsh

cd ..\00_本源\_sp_configs
Copy-Item dao.sample.json vm-甲.json

# 部署时引用此件

```

#### 2.4.3 三层覆 (优先序)

```text
1. process.env.SP_*      ★ 最高优 (主公本机 ENV · 兼容老法)
2. cli --sp-* 参         (一字覆 · 不立 json)
3. _sp_configs/{name}.json (主公编件 · 长期态)
4. 默 bypass              (无配置时之安全态)
```

## 五 · Playbook 框架 (印 122 立 · Cognition 官认)

`00_本源/playbook_helper.js` · 0 deps · 实 **Cognition Playbook** 框架：

```text
Playbook = 主公预定 workflow (e.g. "代码审计 · 分 5 步")
         + 触发条件 (e.g. message 含 'audit')
         + 执行链 (each step 之 SP override + post-process)
         + state 持久 (本地 .dao/playbooks.json)
```

### 5.1 配 (本地 JSON · 主公自编)

`%USERPROFILE%\.dao\playbooks.json`：

```json
{
  "playbooks": [
    {
      "name": "code_audit",
      "trigger": { "match": "(?i)audit|审计" },
      "steps": [
        { "sp_override": "你是代码审计专家 · 第一步识别架构", "max_tokens": 500 },
        { "sp_override": "第二步识别风险点 · 列 top 5",        "max_tokens": 800 },
        { "sp_override": "第三步给修复建议 · 按风险序",         "max_tokens": 1000 }
      ]
    }
  ]
}
```

### 5.2 用

```python
client = OpenAI(
    base_url=f"{vm_url}/port/7780/v1", api_key="...",
```

## 六 · response 之 sp meta (主公 client 自适)

```json

  "id": "chatcmpl_xxx",
  "object": "chat.completion",

      "daemonSource": "silk_chapter_22"

```

**主公 client 一字读**:

```python
resp = client.chat.completions.create(...)

print(resp.x_dao.sp.daemonSource)    # "silk_chapter_22"
```
