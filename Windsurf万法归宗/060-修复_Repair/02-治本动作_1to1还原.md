# 02 · 治本动作 · 1:1 还原 bak_predao

## 治之径

```
1. 杀全 Windsurf + language_server (含 ghost)
   ↓
2. 备份当前被改 extension.js (回滚保险)
   ↓
3. Copy-Item bak_predao_1778654701851 → extension.js (1:1 还原)
   ↓
4. 验·新 ext.js 真无 :8878 + 真含 server.codeium.com
   ↓
5. schtasks 主公 session 净启 Windsurf
   ↓
6. 等 WAM 自然注入 (90s)
```

## 完整 PowerShell 命令

```powershell
Invoke-Command -ComputerName 192.168.31.179 -ScriptBlock {

    $ts    = (Get-Date).ToString('yyyyMMdd_HHmmss')
    $extJs = 'E:\Windsurf\resources\app\extensions\windsurf\dist\extension.js'
    $bak   = 'E:\Windsurf\resources\app\extensions\windsurf\dist\extension.js.bak_predao_1778654701851'

    # === Step 1 · 彻杀全 ===
    $before = Get-Process -Name Windsurf,language_server* -EA SilentlyContinue
    Write-Output ('关前: ' + $before.Count + ' 进程')
    $before | ForEach-Object { try { Stop-Process -Id $_.Id -Force -EA Stop } catch {} }
    Start-Sleep 4
    # 二次硬杀残 ghost
    $stragglers = Get-Process -Name Windsurf,language_server* -EA SilentlyContinue
    if ($stragglers) {
        $stragglers | ForEach-Object { try { Stop-Process -Id $_.Id -Force -EA Stop } catch {} }
        Start-Sleep 3
    }

    # === Step 2 · 备份当前被改 (回滚保险) ===
    $bakCur = "E:\Windsurf\resources\app\extensions\windsurf\dist\extension.js.bak_8878hack_$ts"
    Copy-Item $extJs $bakCur -Force

    # === Step 3 · 1:1 还原 bak_predao ===
    Copy-Item $bak $extJs -Force

    # === Step 4 · 验新 ext.js ===
    $newContent = Get-Content $extJs -Raw
    if ($newContent.IndexOf('127.0.0.1:8878') -ge 0) {
        Write-Output '⚠ 还原失败·新 ext.js 仍含 :8878'
        return
    }
    if ($newContent.IndexOf('server.codeium.com') -lt 0) {
        Write-Output '⚠ 新 ext.js 不含 server.codeium.com'
        return
    }
    Write-Output '✓ 新 ext.js 真无 :8878 + 真含 server.codeium.com'

    # === Step 5 · schtasks 净启 ===
    $tn = 'WSReviveRestored'
    schtasks /Delete /TN $tn /F 2>$null | Out-Null
    $ft = (Get-Date).AddSeconds(20).ToString('HH:mm')
    schtasks /Create /TN $tn /TR '"E:\Windsurf\Windsurf.exe"' `
             /SC ONCE /ST $ft /RU zhouyoukang /IT /F | Out-Null
    schtasks /Run /TN $tn | Out-Null

    # === Step 6 · 等启完 ===
    Start-Sleep 90
    schtasks /Delete /TN $tn /F 2>$null | Out-Null
    Write-Output '✓ 治本完成'
}
```

## 关键设计点

### 为何用 schtasks 而非 explorer.exe / Start-Process

PowerShell Remote 之 session 与主公 RDP session 是不同 Windows session (session 0 vs session 2)。直接 `Start-Process Windsurf` 会启在 session 0 之 service context·主公 RDP 中看不到。

`schtasks /RU zhouyoukang /IT` 之 `/IT` flag 让 task 启入主公 (zhouyoukang) 之 interactive session 2·主公 RDP 内可见。

### 为何要 5 秒等待 + 二次硬杀

某些 Windsurf 子进程 (NodeService / language_server / ext-host helper) 终止时持有 `Roaming\Windsurf` 目录之文件锁 (state.vscdb / SingletonLock 等)·杀主进程后这些子进程可能延迟 1-3 秒才彻底释放锁。

5 秒等待 + 二次扫漏杀 = 保证下次启 Windsurf 时不会因锁占用而进 zombie/half-init 态。

### 为何不用 Stop-Process -Force 一招

某些 Windsurf 进程在被 Force 杀时会触发 Windows Error Reporting (WER) 弹 `Windsurf.exe 已停止工作` dialog·阻塞 ext-host 重启。逐个 try/catch + EA Stop 是为优雅。

### 为何还原 bak_predao 而非另装新 Windsurf

- 保留主公现有 user-data-dir (workspaceStorage / globalStorage / extensions)
- 保留 WAM (devaid.rt-flow-2.7.5) 等三方扩展 + accounts.md
- 保留主公的 Settings/Keybindings 等个人配置
- 仅替换 9.18MB 之 binary 文件 — 最小侵入·一键回滚
