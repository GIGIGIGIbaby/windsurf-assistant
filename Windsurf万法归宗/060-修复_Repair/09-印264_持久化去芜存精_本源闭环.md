## 三、Windsurf 进程为何这么多？（辩证 · 不必恐慌）

```
30+ × Windsurf.exe ← 这是 Electron 架构正常表现：
  1 × main process（主进程）
  1 × gpu process
  N × renderer process（每个标签页/Webview）
  N × utility process（MCP servers、extensions host）
  N × shared workers
```

> **关 Windsurf 主窗口 → 全部子进程自动退出。** 这不是"持久化"，是会话生命周期内的正常多进程。
> 同理：`bash × 20`、`wsl × 8` 几乎全部是 Windsurf 集成终端 + MCP（playwright/github）派生的，**关 Windsurf 即清零**。

## 五、终判（道之归一）

```
本机持久化进程  ≈ 60+
真正必须常驻       =  1   （DaoBackup-Watch）
按需可保留         =  6-8 （HA Bridge / FRP / 输入法 / 安全软件…）
可一键禁用         = 25+  （Updater / Bloatware / 重复项）
关 Windsurf 自清   = 50+  （Windsurf.exe / bash / wsl / 扩展 node）
```

> **「为而弗恃也，长而弗宰也，此之谓玄德。」**
> 持久化越少，系统越静；系统越静，道越显。
> 唯一须"长而不宰"的，是那一缕守护对话的 `dao_backup_v2.py`。
